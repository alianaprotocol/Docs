## 1. 愿景与核心价值 (Vision) 🚀

**Aliana Protocol** 是一场关于去中心化金融（DeFi）高频流动性的社会实验。我们致力于解决传统 DeFi 协议中存在的“回本周期长”、“资金利用率低”以及“通胀失控”的三大痛点。

通过引入 **ALI** 作为核心通证，Aliana 构建了一个自我调节的闭环经济系统：
1.  **即时满足**: 通过预付利息，让投资者在入场的瞬间即获得回报。
2.  **幸运通缩**: 通过赎回燃烧费（Burn Fee），迫使短视资金为长期持有者买单，实现代币通缩。
3.  **公平分发**: 通过去中心化的多层级激励网络，将营销预算直接回馈给社区建设者。

## 2. 核心理财协议详解 (Investment Protocol) ⚙️

Aliana 的理财模型设计精巧，采用 **ALI 本位** 进行结算。所有合约参数均已固化在链上，不可篡改。

### 2.1 阿里投资计划 (ALP Plans) 📊

协议提供两种不同周期的理财计划，分别针对**高频体验型**用户和**稳健收益型**用户。

| 特性 | **ALP-1 (体验版)** | **ALP-2 (进阶版)** |
| :--- | :--- | :--- |
| **适用人群** | 新手体验、短期资金周转 | 长期持有者、大资金理财 |
| **最低存款** | **10 ALI** | **300 ALI** |
| **锁仓周期** | **7 天** | **15 天** |
| **日化收益 (Daily ROI)** | 1.00% | 1.50% |
| **周期总收益 (Total ROI)** | **7.00%** | **22.50%** |
| **推荐策略** | 快速复投，刷流水 | 长期锁定，获取高额利息 |

### 2.2 核心创新：预付利息机制 (Prepaid Interest) ⚡️

这是 Aliana 最具颠覆性的创新。在传统理财中，利息是随时间线性释放的；而在 Aliana，**利息是前置发放的**。

**执行流程：**
1.  用户向合约存入本金（例如 10,000 ALI）。
2.  合约立即计算该周期的所有利息（例如 Plan 1 为 700 ALI）。
3.  **在同一笔交易中**，合约将本金锁定，并将 700 ALI 利息**立即转回**用户钱包。
4.  用户实际锁仓风险仅为：`本金 - 预付利息`。

> **优势**: 极大地提高了资金周转率，用户可以用预付的利息立即进行复投（Compound）或变现，实现“利滚利”的加速效应。

### 2.3 退出机制：燃烧费与通缩模型 (Burn Fee) 🔥

为了维持 ALI 代币的价值稳定，协议在资金**流出**环节设置了燃烧损耗。当锁仓期结束，用户申请赎回本金（Redeem）时，系统会根据用户的**忠诚度（复投次数）**扣除一定比例的本金并直接销毁。

**复投次数 (Reinvest Count) 与燃烧费率表：**

| 复投次数 | 燃烧费率 (Burn Fee) | 说明 |
| :--- | :--- | :--- |
| **0 次** | **4%** | "快进快出"的惩罚性费率 |
| **1 次** | **3%** | 初步参与 |
| **2 次** | **2%** | 进阶参与 |
| **3 次及以上** | **1%** | **忠诚用户最优费率** |

*注：复投操作本身（Reinvest）**不扣除**燃烧费，仅在最终提现（Redeem）时扣除。*

## 3. 市场激励网络 (Referral Network) 🤝

Aliana 鼓励社区裂变，并将大部分利润分配给推广者。所有奖金均由**智能合约自动分账**，无需人工审核。

### 3.1 20层代数奖励 (Unilevel Rewards) 🪜

这是一项基于**新增业绩 + 复投（存款本金）**的直接奖励。

*   **宽度**: 无限宽，你可以推荐无数个直推。
*   **深度**: 最深可达 20 层。

| 层级 (Level) | 奖励比例 | 解锁条件 (Unlock Requirement) |
| :--- | :--- | :--- |
| **L1 (直推)** | **10%** | 个人存款 ≥ 10 ALI |
| **L2** | **5%** | 个人存款 ≥ 10 ALI |
| **L3** | **3%** | 个人存款 ≥ 10 ALI |
| **L4 - L5** | **1%** | 个人存款 ≥ 10 ALI |
| **L6 - L10** | **1%** | 需 2 个合格直推部门 (Active Directs) |
| **L11 - L15** | **0.5%** | 需 3 个合格直推部门 |
| **L16 - L20** | **0.25%** | 需 5 个合格直推部门 |

> *定义: "合格直推"指该下级账户当前的活跃投资额 ≥ 100 ALI。*

### 3.3 平级与越级机制 (Override Mechanism) 🚀

传统级差制中，下级超越上级会导致上级收益归零。Aliana 引入了**Override Bonus** 解决此问题。

*   **触发条件**: 你的等级 ≥ **A3**，且下级等级 ≥ 你的等级。
*   **奖励内容**: 你将获得该下级所领取的级差奖金总额的 **10%** 作为额外奖励。
*   **意义**: 鼓励上级培养强大的下级，消除"被超越"的恐惧。

### 3.4 社区领袖超级节点 (Community Leader) 👑

由项目方或 DAO 投票选出的特殊贡献者（如KOL、社区合伙人）。
*   **权益**: 绑定在超级节点下的所有用户（无限代），其每一笔存款的 **2%** 和每一笔利息的 **5%** 将直接奖励给超级节点。

## 4. 游戏化幸运系统 (Gamification) 🎲

为了增加系统的趣味性和粘性，Aliana 内置了多重幸运机制。

### 4.1 每日 FOMO 奖池 (Daily Jackpot) 🔥

每日全网新增存款的 **3%** 会自动流入 FOMO 奖池。该奖池每 24 小时（UTC 00:00）结算一次。

### 4.2 奖池分配规则 (Distribution) 📊

在 V2 版本中，我们引入了更加公平的“推广权重”机制，取代了容易被机器人垄断的“最后幸存者”机制。

| 奖项 | 占比 | 描述 |
| :--- | :--- | :--- |
| **Top 3 巨鲸奖** | **20%** | 当日个人存款总额最高的前三名用户。<br>🥇 **50%** (Top 1)<br>🥈 **30%** (Top 2)<br>🥉 **20%** (Top 3) |
| **幸运锦鲤奖** | **30%** | 系统通过链上随机数（VRF）抽取 **10 名** 当日存款 ≥ 10 ALI 的用户，均分这部分奖金。 |
| **推广增长奖** | **20%** | **(V2 新增)** 奖励当日推广业绩最活跃的用户。根据“有效直推人数”和“直推业绩”计算权重进行加权分配。 |
| **奖池滚存** | **30%** | 剩余资金自动滚入下一期奖池，使奖池规模不断滚雪球，确保持续吸引力。 |

> **注意**: FOMO 奖金发放方式分为两种：
> 1. **Top 3 巨鲸奖**与**幸运锦鲤奖**：自动发送至用户钱包，无需消耗配额。
> 2. **推广增长奖**：发放至用户的 `fomoBonuses` 账户中，用户需手动执行 Claim 操作，并消耗等额的 Bonus Quota。

### 4.3 配额系统：泡沫抑制器 (Bonus Quota) 🎫

为了防止动态奖金产生的泡沫过大，所有推广奖励的领取都受限于**“配额”**。

*   **规则**: `1 ALI 奖金 = 消耗 1 ALI 配额`。没有配额，无法领取奖励。
*   **获取方式**:
    1.  **首充赠送**: 首次存款自动获得 **300 ALI** 配额。
    2.  **购买/销毁**: 用户可以通过销毁 ALI 代币来购买更多配额（比例通常为 1:3，即销毁 1 ALI 获得 3 配额），这进一步促进了通缩。

## 5. P2P 场外交易市场 (P2P Exchange) 💹

为了解决 ALI 代币与 USDT 等主流资产的兑换需求，Aliana 协议内置了一个去中心化的 P2P 场外交易引擎。该引擎完全由智能合约托管，消除了中心化交易所（CEX）的信任风险。

### 5.1 核心机制 (Mechanism) ⚙️

交易市场采用 **FIFO (First-In-First-Out, 先进先出)** 队列匹配机制，确保交易的公平性。

1.  **挂单 (Maker)**: 用户发布买单或卖单，资产被锁定在合约中。
2.  **吃单 (Taker)**: 用户根据当前队列顺序，匹配并成交挂单。
3.  **队列排序**: 
    *   按照挂单时间戳严格排序，先挂单者先成交。
    *   支持部分成交 (Partial Fill)，未成交部分继续保留在队列头部。
    *   **懒惰清理 (Lazy Cleanup)**: 系统自动跳过已过期或已取消的无效挂单，无需人工干预。

### 5.2 交易规则 (Trading Rules) 📜

*   **固定汇率**: 为防止价格剧烈波动，系统采用固定汇率制度（1 ALI = 1 USDT）。
    *   校验公式: `卖出数量 × 买入单位价格 = 买入数量 × 卖出单位价格`
*   **手续费 (Trading Fee)**:
    *   **费率**: **2%** (由 **卖出 ALI** 的一方支付)。
    *   **规则**: 无论是挂单方 (Maker) 还是吃单方 (Taker)，只要是卖出 ALI 换取其他资产，均需支付手续费。买入 ALI 不收费。
    *   **支付方式**: 从成交获得的 USDT/ETH 中自动扣除。
*   **资金安全**:
    *   所有挂单资产均托管在合约中，Maker 可随时取消未成交的订单并全额退款。

### 5.3 极致体验：Gasless 挂单 (Permit) ✨

为了提升交易体验，我们集成了 **EIP-2612 Permit** 标准。
*   **传统模式**: 挂单需要两步交易（1. Approve 授权代币 -> 2. Create Order 挂单）。
*   **Aliana 模式**: 仅需一步！
    1.  用户在钱包中对挂单金额进行签名（链下操作，无 Gas）。
    2.  直接发起挂单交易，合约自动验证签名并完成代币扣款。
    3.  **结果**: 节省 50% 的操作步骤和 Gas 费用。
