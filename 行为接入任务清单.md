# Aliana Protocol - Web2 行为接入任务清单 (Web2 Integration Tasks)

在 **混合积分系统 (Hybrid Points System)** 架构下，我们可以将大量无需上链的 Web2 行为转化为积分任务。这不仅能极大地降低用户参与门槛（0 Gas），还能作为强大的运营抓手，灵活引导用户行为。

本清单列举了可快速接入的高价值 Web2 行为场景。

---

## 1. 社交裂变类 (Social Growth)
*目标：提升项目在公域流量（Twitter）和私域流量（Discord/TG）的曝光度与活跃度。*

| 任务名称 | 行为描述 | 验证方式 | 建议积分 | 频率 |
| :--- | :--- | :--- | :--- | :--- |
| **Follow X** | 关注官方 Twitter 账号 | Twitter API 验证 | **50 pts** | 一次性 |
| **Daily RT** | 转发/引用当日置顶推文 | Twitter API 检测 | **20 pts** | 每日 |
| **Name Suffix** | 推特昵称添加后缀 (e.g., `💙 Aliana`) | 每日定时快照检测 | **10 pts** | 每日 |
| **Join Discord** | 加入官方 Discord 服务器 | Discord OAuth2 | **50 pts** | 一次性 |
| **Chat Active** | 在 Discord 综合频道发言 ≥3 句 | Discord Bot 监听 | **5 pts** | 每日 |
| **Inviter** | 生成链接邀请 1 名真人进群 | Discord Bot 统计 | **20 pts** | 无上限 |

## 5. 实施建议

1.  **循序渐进**：
    *   **P0 (首发)**：Twitter 关注/转发、Discord 加入、绑定邮箱。这些接口成熟，开发成本低。
    *   **P1 (进阶)**：每日签到、阅读白皮书。
    *   **P2 (运营)**：UGC 审核、问卷调查。

2.  **防刷机制**：
    *   **Twitter**：要求推特账号注册时间 > 3个月，粉丝数 > 10。
    *   **Discord**：要求绑定手机号。
    *   **IP 限制**：同一 IP 每日只能完成 3 个账号的任务。

3.  **动态调整**：
    *   后台应具备配置能力，可随时调整任务的分值。例如项目发新推文时，临时将“转发任务”积分翻倍，瞬间拉满热度。
